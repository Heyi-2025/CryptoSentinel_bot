# CryptoSentinel Project Context
- **项目定位:** 基于 Telegram 的高频加密资产异动监控系统。
- **技术栈:** Python 3.10+, 纯异步 (asyncio), SQLite (aiosqlite), PM2。
- **核心架构 (解耦设计):**
  1. `db_manager.py`: 负责数据库的增删改查。
  2. `market_engine.py`: 生产者。轮询 OKX/Binance 等交易所，计算 pandas_ta 指标，将最新一根 K 线结果覆盖写入本地 `cryptosentinel_cache.json`。
  3. `notifier.py`: 消费者。监控 cache 变化，比对数据库订阅名单，调用 TG 接口推流。
  4. `bot.py`: 交互前端。纯 UI 逻辑，负责把用户的点单存入数据库。
- **开发原则:** - 绝对不要在主循环中使用阻塞式 `time.sleep`，必须用 `asyncio.sleep`。
  - 交易所请求必须加入异常处理和限流休眠 (Rate Limit)。
  - 代码需包含详细的中文注释。
